The Amplification Paradox in Recommender Systems

Recommendation systems are a crucial part of today's digital world, connecting users with content that aligns with their interests. However, there is an ongoing debate about the role these algorithms play in promoting extreme content, which can negatively affect users and society.

Previous results on whether recommender systems drive users toward extreme content are mixed. Algorithmic audits of recommender systems suggest they do, but analyses of real user traces suggest they do not. This blog post (and its accompanying paper) provide a possible explanation for the apparent paradox: algorithmic audits that do not model user preferences can be misleading.

A simple recommender system

Key to modern recommender system is the notion of collaborative filtering: recommending items to users based on other, similar users. We demonstrate the inner workings of a simple recommender system using an interactive matrix of users (rows) and items (columns). Each cell (i, j) indicates whether user i has consumed item j .

A simple agent-based model

Using an agent-based model, we study recommender systems like the one we just showed you. Agents in the model select videos from the suggestions provided by the recommender, either at random (like bots in algorithmic audits) or following some utility function (that can mimic how actual people behave). We consider the typical left-right political spectrum, dividing content in our recommender system into five topics: Far Left, Left, Center, Right, and Far Right. Further, we create four “personas,” each of which is keener on content aligned with one of the topics.

Random selection

With the agent-based model, we can see what categories are recommended when users select items "at random." This is similar to how many algorithmic audits are performed. You can select a seed item (that is, what is the first video the user departs from) nd see how many videos of each category they will consume in an algorithmic session (we consider a session of length 20). Here, we find that the prevalence of far-right and far-left content increases as users engage in longer algorithmically driven sessions.

Utility-based selection

Next, we simulate what categories are recommended when users choose items according to some utility function. Here, however, we consider a counterfactual, shown as the grey box in the animation below, which is the "relative utility" of each category to each persona. The relative utility is how much users would consume from each category if they choose items at random, with replacement, and with probability proportional to the utility items have for them. Here, we find that extreme content is recommended less that it would be in the absence of the recommender system, being thus deamplified. This suggests a fundamental limitation with existing audits.
